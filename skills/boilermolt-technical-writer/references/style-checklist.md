# style checklist

Before finalizing, check:

- [ ] Audience is explicit or inferable
- [ ] Active voice throughout
- [ ] No filler language
- [ ] Terms are consistent
- [ ] Steps are actionable and ordered
- [ ] Validation is concrete
- [ ] Rollback/failure behavior covered (if applicable)
- [ ] Metadata present for governance/product/ops docs
- [ ] Links/paths are valid
